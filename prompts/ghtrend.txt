GitHub の AI 関連トレンドリポジトリを調査し、結果をこのチャンネルに投稿してください。

## 期間ルール（最重要）
実行時刻（JST）を起点に、以下の条件を満たす repo だけを採用する。
- 直近 7 日以内に最新コミット (pushed_at) または star 増加があった repo のみ
- 直近 24h に急上昇（デイリー trending 掲載 / star の急な増加）があった repo は ⭐ マークで優先表示
- 7 日以上前からアップデートが無い repo、活動実績が取得できない repo は採用しない

## 手順
1. 最初に実行時刻の JST 年月日時を宣言する。必ず `date '+%Y-%m-%d %H:%M JST'` をシェル実行して得た値を使う。
   - 7 日ウィンドウ開始 = 宣言した実行日の 7 日前 00:00 JST
   - 24h ウィンドウ開始 = 宣言した実行日の 1 日前 00:00 JST
2. WebSearch で以下を検索。クエリに 7 日ウィンドウ開始日以降の `after:YYYY-MM-DD` を付与する：
   - "github trending AI agent this week"
   - "github trending machine learning this week"
   - "github stars AI framework 2026"
3. 候補 repo ごとに GitHub 本体（可能なら公式 API / 公式ページ）で以下を確認：
   - リポジトリ名（owner/name）
   - スター数（現在値）
   - 最終コミット日 (YYYY-MM-DD, pushed_at)
   - 直近 24h での動き: 急上昇 / Trending 掲載 / 通常
   - 説明（日本語訳）
   - 言語
   - カテゴリ: Agent / LLM / Tool / Infra / Research
   - リポジトリURL（https://github.com/owner/name）
   - 関連リンク（公式ドキュメント・デモサイト・論文など、あれば）
4. 採否判定：
   - 最終コミット日が 7 日ウィンドウ内 → 採用
   - それ以外 → ドロップ（出力に含めない）
   - 直近 24h に動きあり → タイトル冒頭に ⭐ を付ける
5. 以下のフォーマットで投稿（Slack mrkdwn）：
   ---
   🐙 GitHub AI トレンド（YYYY-MM-DD HH:MM JST）
   採用期間: 直近 7 日以内に活動のある repo | ⭐ = 直近 24h の急上昇

   1. ⭐ owner/name — ★xx,xxxk
      説明（日本語）
      言語: xx | カテゴリ: xx | 最終commit: YYYY-MM-DD
      🔗 https://github.com/owner/name
      📎 関連: [公式ドキュメントURL等あれば]

   （5〜10 件を上記フォーマットで続ける）

   ---
   収集サマリー
   • 検索キーワード: 3 件
   • 採用 repo: N 件（うち ⭐ 24h 急上昇: a 件）
   • 期間外ドロップ: M 件
   • 実行時刻: YYYY-MM-DD HH:MM JST（step 1 で宣言した値）
   ---
