以下の手順で「直近の AI ニュース」を収集し、結果をこのチャンネルに投稿してください。

## 期間ルール（最重要）
実行時刻（JST）から逆算して、以下の期間内に公式な発表日・公開日があるものだけを採用する。
- 重要度 4-5（業界変革・主要プレイヤー）: 直近 72 時間以内
- 重要度 1-3（注目技術・一般・参考）: 直近 24 時間以内
発表日が不明な記事、期間外の記事は採用しない（途中で見つけても最終出力に含めない）。
「昨日」「先週」等の相対表現は実行日を基準に絶対日付 (YYYY-MM-DD) に変換して判定する。

## 手順
1. 最初に実行時刻の JST 年月日時を宣言する。必ず `date '+%Y-%m-%d %H:%M JST'` をシェル実行して得た値を使うこと（cron 予定時刻ではなく、実コンテナ起動時刻を採る）。
   - 72h ウィンドウ開始 = 宣言した実行日の 3 日前 00:00 JST
   - 24h ウィンドウ開始 = 宣言した実行日の 1 日前 00:00 JST
2. WebSearch で以下のキーワードを検索（日本語・英語 各 3 件）。
   クエリには 72h ウィンドウ開始日以降の `after:YYYY-MM-DD` を付与する。
   - "AI agent 2026"
   - "生成AI 最新"
   - "LLM new release"
   - "MCP protocol AI"
   - "AIエージェント フレームワーク"
3. 各記事について以下を整理：
   - タイトル（原文）
   - 発表日 (YYYY-MM-DD)  ← 公式発表日 / 公開日 / 記事冒頭の日付から特定。判別不能なら除外
   - 日本語要約（3 行以内）
   - カテゴリ: LLM / Agent / Tool / Policy / Research / Product / Funding / OSS / Infra / Ethics
   - 重要度: 1-5（5=業界変革、4=主要プレイヤー、3=注目技術、2=一般、1=参考）
   - 一次情報URL（公式発表・論文・プレスリリース・公式ブログ。見つからない場合は報道記事URL ＋「※報道」）
   - ソースURL（情報取得元）
   - 地域: us / eu / jp / cn / global
4. 採否判定：
   - 重要度 4-5 かつ 発表日が 72h ウィンドウ内 → 採用
   - 重要度 1-3 かつ 発表日が 24h ウィンドウ内 → 採用
   - 上記いずれでもない → ドロップ（出力に含めない）
5. 以下のフォーマットで投稿：
   ---
   📰 AI Pulse 収集レポート（YYYY-MM-DD HH:MM JST）
   採用期間: 重要度4-5 → 直近72h / 重要度1-3 → 直近24h

   【重要度5-4】
   🔴 [カテゴリ] タイトル
   発表日: YYYY-MM-DD
   要約: ～～～
   一次情報: ～～～
   ソース: ～～～
   地域: xx | 重要度: N

   【重要度3以下】
   🟡 [カテゴリ] タイトル（1行要約）
   発表日: YYYY-MM-DD
   一次情報: ～～～
   ソース: ～～～

   ---
   収集サマリー
   • 検索キーワード: 5 件
   • 採用記事: N 件（重要度4-5: a 件 / 重要度1-3: b 件）
   • 期間外ドロップ: M 件
   • 実行時刻: YYYY-MM-DD HH:MM JST（step 1 で宣言した値）
